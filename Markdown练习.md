# Geek精神

1.在申请加入到Geek组之前，我搜索过**Geek**这个英文单词，词典告诉我，它的原义为*怪才、痴迷于某领域的人*，我想，也许**热爱**就是Geek精神的一大部分。

2.通过对发表在第一次任务之前的**写在前面的话**[原网址][1]的学习，我能感受到：Geek并非止步于热爱，他们更是严谨的实践者，是对硬核技术的攻克者，所以我想，Geek的精神，应当总结为：**无限热爱和严谨实践的结合**。

# Github学习过程

1.通过在bilibili上查找的科普视频[原视频][2]进行第一时间的学习，我了解到Github是比我想象的运用范围广得多的代码托管与协作平台，它汇聚了全球开源项目、支持的不只是代码的运营，更是一本内容无比丰富的百科全书。

2.在Github中存在着核心协作术语以实现它的核心功能之一：**协作开发**

以下是两个核心词的释义：

| Fork                                                       | PR                                                   |
| ---------------------------------------------------------- | ---------------------------------------------------- |
| 将他人Repository完整拷贝到自己的Github账号下，形成独立副本 | 在自己Fork的仓库中修改后向原仓库维护者提出*合并请求* |

这是由我在浏览网页时发现了不懂的单词加上任务提示，而后在夸克上搜索后得到的结果。

# 在进入Github时遇到的阻碍

## 阻碍的发现

在尝试直接通过链接进入Github时，我发现我完全进入不了，在尝试多次后，突然又发现能够进入了。

## 阻碍的解决

通过询问我的mentor，我了解到Github的稳定运行需要加速器来实现，否则就会成为概率事件，于是我在网上搜索了稳定的加速器下载方式，通过操作成功购买了加速器，实现了Github的稳定运维。
